# 蛋蛋量化交易策略 v1.0

## 账户信息
- 初始资金：¥50,000
- API地址：http://localhost/api
- 单票上限：¥10,000（总仓位20%）

---

## ⚕️ 交易前必读：查阅历史复盘

**每次下单前，必须先查阅历史复盘记录！**

1. 调用 `/api/daily` 获取历史复盘
2. 回顾最近5次交易记录 `/api/trades`
3. 检查是否有重复失误、是否有可复制的成功模式
4. 根据历史调整当日策略

**进化机制：**
- 每次复盘记录盈亏和操作理由
- 分析成功/失败原因
- 优化策略参数（RSI阈值、止盈点等）
- 避免重复同样的错误

---

## 一、建仓规则（必须全部满足）

**指数条件：**
- 上证指数必须站稳5日均线（MA5 > MA10，且价格 > MA5）
- 创业板指数同样要求

**个股条件（满足其一即可建仓）：**
1. **均线金叉** — MA5上穿MA10，且价格站上MA5
2. **RSI超卖反弹** — RSI < 35，且开始拐头向上
3. **放量突破** — 成交量 > 2倍5日均量，且涨幅 > 2%

**仓位规则：**
- 每只股票最多投入 ¥10,000
- 总仓位不超过 20%（即 ¥10,000）
- 同时持仓不超过 2只

---

## 二、持仓管理

### 止损规则
- 个股亏损达到 **-8%** → 立即止损
- 指数跌破5日线且未收回 → 减仓50%

### 止盈规则
- 盈利达到 **+10%** → 止盈1/3
- 盈利达到 **+15%** → 再止盈1/3
- 盈利达到 **+20%** → 清仓

### 动态止盈
- 盈利>10%后，如果回撤超过最高点的 **-5%** → 清仓

---

## 三、卖出条件（满足任一即卖出）

1. 个股RSI > 70（超买）
2. 价格跌破MA5且收不回来
3. 放量滞涨（涨幅<1%但成交量暴增）
4. 触发止损/止盈规则

---

## 四、盘中监控频率

| 时间段 | 检查频率 | 重点 |
|--------|----------|------|
| 09:30-09:45 | 每5分钟 | 开盘异动 |
| 09:45-11:30 | 每10分钟 | 趋势确认 |
| 11:30-13:00 | 不检查 | 午休 |
| 13:00-14:55 | 每5分钟 | 尾盘决策 |
| 14:55-15:00 | 每分钟 | 收盘操作 |

---

## 五、仓位管理原则

1. **宁错过，不做错** — 不符合规则坚决不动
2. **分批建仓** — 首次建仓50%，确认趋势再加50%
3. **保留子弹** — 始终保留30%现金在手
4. **顺势而为** — 指数在5日线下，禁止建仓

## 六、复盘记录保存

**每次复盘必须包含：**
- 今日账户盈亏（总市值 - 初始资金）
- 持仓情况（每只股票的现价、成本、盈亏%）
- 交易记录（买入/卖出的股票、时间、价格、理由）
- 操作反思（这次操作对在哪/错在哪）
- 明日策略调整（根据今天学到的调整）

**调用API保存：**
```bash
# 复盘通过 POST /api/review 保存
curl -X POST http://localhost/api/review \
  -H "Content-Type: application/json" \
  -d '{"content": "复盘内容", "tags": ["标签"], "strategies": ["策略"]}'
```

**不复盘=没记录！必须写进系统！**
