## ⚠️ A股交易规则（必须遵守！）

### 交易时间
- 集合竞价：9:15-9:25（可挂单，不可成交）
- 开盘竞价：9:25-9:30
- **连续竞价：9:30-11:30 / 13:00-15:00**
- 收盘定价：14:57-15:00

### 交易单位
- 最少**100股（1手）**
- 每次买卖必须是100股的整数倍

### 涨跌幅限制
- 普通股票：±10%
- ST股票：±5%

### 手续费（实际）
| 类型 | 费率 | 收取时机 |
|------|------|----------|
| 印花税 | 0.05% | **仅卖出时** |
| 过户费 | 0.001% | 仅沪市 |
| 券商佣金 | ~0.03% | 买卖均收 |

### T+1制度
- **当日买入，下一交易日才能卖出**

## ⚕️ 交易前必读：查阅历史复盘

**每次下单前，必须先查阅历史复盘记录！**

1. 调用 `/api/daily` 获取历史复盘
2. 回顾最近5次交易记录 `/api/trades`
3. 检查是否有重复失误、是否有可复制的成功模式
4. 根据历史调整当日策略

**进化机制：**
- 每次复盘记录盈亏和操作理由
- 分析成功/失败原因
- 优化策略参数（RSI阈值、止盈点等）
- 避免重复同样的错误

## 一、买入规则

### 买入时间窗口
- **14:50 - 14:55**（尾盘最后5分钟，价格更稳）

### 个股条件（全部满足）

| 条件 | 要求 |
|------|------|
| 涨幅 | **3% - 5%** |
| 成交量 | **> 1.5倍** 5日均量 |
| 换手率 | **3% - 10%** |
| 流通市值 | **50亿 - 200亿** |
| RSI | **40 - 65**（不超买不超卖） |
| 分时均价线 | 当前价格 **站上** 分时均价线 |
| 强于大盘 | 个股涨幅 **>** 大盘涨幅 |

---

## 二、卖出规则（次日早盘冲高即走）

### 核心原则
- **一夜持股，次日了结**
- 不贪不恋，冲高就跑

### 卖出时机
- **次日09:30 - 10:30** 冲高卖出
- 若开盘直接低开，观察15分钟内是否有反弹，无反弹立即止损

### 卖出条件（满足任一即卖出）

| 条件 | 操作 |
|------|------|
| 次日冲高 **+5%~8%** | 止盈卖出 |
| 次日冲高回落超 **-2%**（从最高点） | 立即卖出 |
| RSI > 70 | 卖出 |
| 放量滞涨（涨幅<1%，成交量暴增） | 卖出 |
| 开盘跌破分时均价线且收不回 | 卖出 |

---

## 三、止损规则

- 个股亏损达到 **-2%** → 立即止损（无条件，超短线纪律要狠）

## 四、仓位管理原则（2026-04-17 修订）

### 核心原则
- **候选股符合全部7个条件 → 满仓买入（100%资金）**
- **宁错过，不做错** — 不符合规则坚决不动

### 多股筛选时的择优规则

多个股票同时满足7个条件时，按以下优先级选最优：

| 优先级 | 条件 | 逻辑 |
|--------|------|------|
| 1 | RSI 适中 | 40-65区间，越高越好（动量更强） |
| 2 | 成交量放大 | >1.5倍，越大资金认可度越高 |
| 3 | 强于大盘幅度 | 个股涨幅 - 大盘涨幅，差值越大越强 |
| 4 | 换手率 | 3%-10%区间，偏高更好 |
| 5 | 流通市值 | 偏小更好（弹性空间大） |

**综合评分最高者，满仓进入。**

---

### 买入数量计算
```
可用资金 = 账户当前现金
买入股数 = floor(可用资金 / 股价 / 100) * 100
```
- 必须是100股的整数倍
- 资金允许时尽量用足，不人为压低

### 仓位执行标准
| 场景 | 操作 |
|------|------|
| 7个条件全部满足 | **满仓**（100%资金）买入 |
| 条件不完整 | 空仓，不买 |
| 止损 | -2% 无条件止损，不等反弹 |

### ⚠️ 绝对禁止
- ❌ 分批建仓（超短线不做长线布局）

## 五、复盘记录保存

**每次复盘必须包含：**
- 今日账户盈亏（总市值 - 初始资金）
- 持仓情况（每只股票的现价、成本、盈亏%）
- 交易记录（买入/卖出的股票、时间、价格、理由）
- 操作反思（这次操作对在哪/错在哪）
- 明日策略调整（根据今天学到的调整）

**调用API保存：**
```bash
curl -X POST http://localhost/api/review \
  -H "Content-Type: application/json" \
  -d '{"content": "复盘内容", "tags": ["标签"], "strategies": ["策略"]}'
```

**不复盘=没记录！必须写进系统！**

## 六、多策略系统 v1.0（2026-05-09）

### 系统架构
```
蛋蛋多策略交易系统
├─ [1] 一夜持股法 (overnight)  ¥25,000
│   └─ 尾盘14:50-14:55买入，次日早盘09:30-10:30卖出
├─ [2] 价值投资 (value)       ¥15,000
│   └─ 巴菲特价值分析，PE<15/ROE>15%，中线1-3个月
└─ [3] 趋势跟踪 (trend)       ¥10,000
    └─ 均线金叉+放量突破，持股1-2周
```

### 数据库结构
- `strategies` 表：策略注册与资金分配
- `trades.strategy_id`：交易归属策略
- `positions.strategy_id`：持仓归属策略
- `equity_curve.strategy_id`：净值曲线按策略

### API变更
- 新增 `/api/strategies` — 获取全部策略及统计
- 新增 `/api/strategies/<id>` — 策略详情
- 新增 `/api/strategies/<id>/toggle` — 启用/禁用
- 新增 `/api/strategies/<id>/capital` — 更新资金分配
- 原有API加 `?strategy_id=` 参数过滤

### Cron任务
| 时间 | 策略 | 任务 |
|------|------|------|
| 每日14:50-14:55 | 一夜持股法 | 尾盘选股+买入 |
| 每日09:30-10:30 | 一夜持股法 | 早盘卖出(每5分钟) |
| 每周一09:30 | 价值投资 | 每周价值扫描 |
| 每日10:00 | 趋势跟踪 | 趋势信号扫描 |
| 每日15:30 | 全部 | 复盘报告 |

### 前端调用示例
```
# 获取一夜持股法持仓
GET /api/portfolio?strategy_id=1

# 获取价值投资交易记录
GET /api/trades?strategy_id=2

# 指定策略买入
POST /api/trade
{"action":"buy","stock_code":"xxx","shares":100,
 "reason":"价值投资","strategy_id":2}
```
