2. **净值图黑屏** — x-show="!loading"导致display:none，ECharts在隐藏容器上初始化失败。改用:style绑定height:0;overflow:hidden替代。

4. **底部被截断** — 容器用了min-height但没有overflow。已改为overflow:auto。

### Alpine.js x-show 覆盖 grid 布局的坑（重要教训）
- `x-show` 恢复显示时设 `display:''`（空字符串），浏览器默认回退到 `display:block`
- 内联 style 里的 `display:grid` 会被 Alpine 覆盖
- **解决方案**：用 `:style` 绑定，loading时`display:none`，否则完整写出`display:grid;grid-template-columns:...`

## 14:12 - 15:20 入场动画调优 + 背景粒子修复

### 入场动画文字不清晰
- 用户截图反馈"聚合成的文字不明显"
- 原因：Canvas采样分辨率512×128太低，中文笔画细节丢失
- 逐步调大：
  - 512×128 → 1024×256 → 2048×512 → **4096×1024**
  - 字体：60px → 120px → 240px → **480px**
  - 粒子数：5000 → 8000 → **12000**
  - 粒子大小：0.3~1.3 → **0.8~2.8**
  - 着色器系数：40 → **70**

### 背景粒子看不到（反复调试）
- 用户多次截图反馈"界面背景没有粒子特效"
- **排查过程**：
  1. 最初以为是 opacity 太低（0.3）→ 改到 0.6，无效
  2. 以为是 z-index 问题（bg-canvas z-index:0 被 z-index:1 的主容器遮住）→ 改 z-index:100，无效
  3. 以为是 body 不透明背景遮住 → 改 body transparent，无效
  4. **最终发现根因**：背景粒子代码在入场动画同一个 IIFE 里，如果入场动画的 ShaderMaterial 编译报错，整个脚本中断，背景粒子根本没执行到！

- **最终修复方案**：
  - 背景粒子独立为单独的 `<script>` 标签（不再和入场动画共享 IIFE）
  - 不用 ShaderMaterial，改用简单的 `PointsMaterial`（避免着色器兼容问题）
  - 紫色发光点，500个粒子缓慢下落
  - z-index:100 浮在界面上方，pointer-events:none 不阻挡操作
  - Canvas opacity:0.6

### 重要教训
- **Three.js ShaderMaterial 的 fragment shader 里 `texture2D` 在 WebGL2 下可能需要改为 `texture`**
- **同一个 IIFE 里如果前面的代码报错，后面的代码全部不执行** — 关键功能应该独立 script 标签
- **调试看不见的 WebGL 效果时**：先用极端参数（纯白、巨大粒子、最高 opacity）确认渲染管线是否工作，再调美观
